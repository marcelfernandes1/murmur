# Murmur — build plan

> Working title: **Murmur** (a quiet voice → text app). Rename anytime; it's only referenced in `project.yml`, the bundle id, and folder names.

A native macOS menu-bar app: hold a hotkey → record mic → transcribe locally with Whisper → insert text at the cursor (or copy to clipboard if no text field is focused). Plus a notch animation and a searchable history.

## Tech decisions (locked)
- **Language/UI:** Swift + SwiftUI + AppKit where needed.
- **Project generation:** [XcodeGen](https://github.com/yonaskolb/XcodeGen) (`project.yml` → `.xcodeproj`). Keeps the project text-based and agent-editable. User opens the generated `.xcodeproj` in Xcode to run/sign.
- **Deps (SwiftPM):**
  - [`KeyboardShortcuts`](https://github.com/sindresorhus/KeyboardShortcuts) — global push-to-talk hotkey (key-down/key-up).
  - [`WhisperKit`](https://github.com/argmaxinc/WhisperKit) — local Core ML Whisper, offline, on-device.
  - [`DynamicNotchKit`](https://github.com/MrKai77/DynamicNotchKit) — notch-anchored animated panel.
- **Persistence:** SwiftData.
- **Min target:** macOS 14 (Sonoma). Apple Silicon strongly recommended (WhisperKit/Core ML).

## Prerequisites (user, one-time)
- Xcode 15+ installed.
- `brew install xcodegen`
- Apple Silicon Mac on macOS 14+.

## Permissions the app will request
- **Microphone** (`NSMicrophoneUsageDescription`) — recording.
- **Accessibility** — focused-element detection + synthetic ⌘V insertion.
- (Possibly **Input Monitoring** depending on hotkey behavior.)

## Phase 0 — Scaffolding & toolchain  ✅
**Goal:** a menu-bar app that compiles from the CLI and launches with an icon in the menu bar (no dock icon).
- `project.yml` (XcodeGen) defining the app target, deployment target, SPM deps, Info.plist, entitlements.
- `Info.plist` with `LSUIElement=YES`, usage strings. `Murmur.entitlements`.
- `MurmurApp.swift` — `@main` `MenuBarExtra` with a placeholder menu (About / Quit).
- `AppState.swift` — central `@Observable` state stub.
- **Verify:** `xcodegen generate` succeeds; `xcodebuild -scheme Murmur build` compiles; user runs it → menu-bar icon appears, Quit works.

## Phase 1 — Core transcription loop (de-risk)  ✅
**Goal:** hold hotkey → speak → release → transcript on the clipboard. No fancy UI.
- `HotkeyManager.swift` — KeyboardShortcuts hold-to-record (key-down starts, key-up stops).
- `AudioRecorder.swift` — `AVAudioEngine` mic tap → resample to 16kHz mono Float32 (`AVAudioConverter`); expose level metering.
- `WhisperService.swift` — load WhisperKit model (`base.en` default), transcribe Float32 buffer → text.
- Mic permission request + denied-state handling.
- On release: transcribe → `NSPasteboard` copy. Log result.
- **Verify:** user holds hotkey, speaks, releases → text is in clipboard (paste anywhere to confirm).

## Phase 2 — Smart insert vs. auto-copy  ✅
**Goal:** type into the focused field if editable; otherwise copy + toast (your spec).
- `AccessibilityManager.swift` — request Accessibility; read `kAXFocusedUIElementAttribute`, decide if editable (AXTextField/AXTextArea/AXComboBox/role+settable value).
- `TextInserter.swift` — editable → set clipboard + synthesize ⌘V via `CGEvent`; else → copy + lightweight toast/notification. Handle secure-input fields gracefully (fall back to copy).
- Accessibility onboarding prompt if not granted.
- **Verify:** dictate into TextEdit → text typed in; dictate with nothing focused → copied + toast shown.

## Phase 3 — History  ✅
**Goal:** persistent, searchable list of past transcripts; click to re-copy.
- `Transcript.swift` — SwiftData model (text, createdAt, duration?).
- `HistoryStore.swift` — save on each successful transcription; fetch/delete/search.
- `HistoryView.swift` — list newest-first, search field, click-to-recopy, swipe/delete. Surfaced from the menu-bar menu.
- **Verify:** transcripts persist across relaunch; re-copy and delete work; search filters.

## Phase 4 — Notch animation  ✅
**Goal:** a polished indicator that flows from the notch through the dictation lifecycle.
- `NotchController.swift` — DynamicNotchKit panel; state machine: idle(hidden) → listening → transcribing → done.
- `WaveformView.swift` — live waveform driven by `AudioRecorder` levels.
- `NotchView.swift` — listening waveform, transcribing shimmer/spinner, done checkmark flourish.
- Handle non-notched + external displays (graceful top-center fallback).
- **Verify:** user observes animation during a full dictation; behaves on notched + non-notched screens.

## Phase 5 — Polish & settings  ✅
**Goal:** ship-ready feel.
- `Preferences.swift` + `SettingsView.swift` — shortcut recorder, Whisper model picker (+ download/progress), mic picker, insert-mode toggle, launch-at-login.
- First-run onboarding flow for Mic + Accessibility permissions.
- Latency tuning, empty/short-audio handling, error toasts.
- App icon + assets; final naming pass.
- Notes for signing/notarization if distributing.
- **Verify:** full end-to-end run; settings persist; permissions onboarding is clean.

## Progress log
- **Long-audio truncation, always-on comparison, looser + generalizing learning** — Four fixes from a feedback burst. (1) **Truncation ("it cut off what I said"):** `WhisperService` set `withoutTimestamps = true` for speed, but WhisperKit needs segment timestamps to seek past its 30s window — so >30s dictations were truncated to the first chunk. Now `withoutTimestamps = !(samples > 28s)`: fast path for short clips, timestamps for long ones (+ a temp `/tmp/murmur_audio.log` WHISPER line to confirm in=Ns/outChars). (2) **Cleanup comparison inconsistent:** it only stored `original` when cleanup changed something, so unchanged dictations never appeared. Now `endRecording` captures the raw ASR text and `deliver` always stores it → the comparison window shows one row per dictation (identical columns when nothing changed); view filter unchanged (`original != nil`), copy updated. (3) **Phonetic gate too strict:** loosened `isPlausibleMishearing` (ratio 0.55→0.34, soundex-alone, same-onset, strong-term bypass) so near-misses like clod→Claude learn while John→Michael stays blocked (self-test 10/10). (4) **Generalize learned terms:** new `CorrectionStore.applyPhonetic` — a token that's a likely mishearing (non-dictionary per `NSSpellChecker`, OR ALL-CAPS like Whisper's guesses) and plausibly matches a learned target gets swapped to it, so learning "Whop" also fixes "WAMP"/"WAP"; lowercase real words (map/wrap/cloud) are protected. Verified standalone. `xcodebuild` → BUILD SUCCEEDED. (Edit-watcher + audio diagnostics still on in `/tmp/murmur_learn.log` / `/tmp/murmur_audio.log` until the feature's confirmed.)
- **Flat-waveform root cause + stable signing** — Reported as "mic icon on, no waves" on a fresh build. File-based diagnostics (`/tmp/murmur_audio.log`) in `AudioRecorder` proved the tap fired and buffers arrived (21 callbacks, full frames) but the converted output was **all zeros** — because the default input was a **3-channel** device and `AVAudioConverter`'s N→mono downmix silently emits silence when the device format has no channel layout (aggregate/virtual devices; this Mac has Background Music + Loom installed). Worked "before" only because the old default was a 1-channel mic. **Fix:** downmix to mono ourselves (pick the loudest channel — passthrough for normal mono mics, isolates the mic channel on multi-channel devices), then let the converter do sample-rate only. Verified: rawPeak/rms went from 0.0 → real signal. Also added `peakAmplitude` + a silence guard in `endRecording` ("No mic signal — check input & permission"). **Separately**, fixed the recurring "re-grant Accessibility every build" pain: the app was **ad-hoc** signed (cdhash changes each build → TCC forgets it). Switched `project.yml` to the user's stable **Apple Development** identity (`DEVELOPMENT_TEAM M8KUF8S2L5`, `CODE_SIGN_STYLE Manual`, `CODE_SIGN_IDENTITY` = cert SHA-1, no profile) → `TeamIdentifier=W2F7R5V23Q`, so Accessibility/Microphone grants now persist across rebuilds (one final re-grant needed because the identity changed from ad-hoc). `xcodebuild` → BUILD SUCCEEDED.
- **Mic selector + silent-input detection** — Un-cut the Phase-5 "deliberately cut" mic picker, prompted by a flat-waveform report (mic icon on, bars dead). Root cause was *not* code (capture path unchanged from HEAD) nor a bad default device (default was the built-in mic) — it was a fresh Debug build at a new path lacking Microphone TCC, so macOS fed it **digital silence** (engine runs, all-zero buffers → flat waveform + empty transcripts). Added `AudioDevices` (CoreAudio enumeration of input devices by stable UID + `deviceID(forUID:)`), `Preferences.inputDeviceUID` (nil = system default), and `AudioRecorder.preferredDeviceUID` which routes the engine's input node via `AudioUnitSetProperty(kAudioOutputUnitProperty_CurrentDevice)` before reading the format. Settings gains a "Microphone" section (device Picker + Refresh + permission hint). Also added `AudioRecorder.peakAmplitude` tracking + a guard in `endRecording`: non-empty samples with peak < 0.0005 ⇒ show "No mic signal — check input & permission" instead of silently transcribing zeros (a working mic always has a noise floor, so no false positives on quiet speech). `xcodebuild` → BUILD SUCCEEDED; CoreAudio enumeration verified standalone (lists built-in mic, iPhone mics, BGM/Loom virtuals with UIDs). Device routing via AUHAL CurrentDevice is best-effort (falls back to default on failure).
- **Auto-learn from edits (Wispr's "word learned")** — Implemented the feature deferred in the accuracy pass. After Murmur pastes a transcript, `FieldEditWatcher` snapshots the focused AX element (`kAXValueAttribute` + caret via `kAXSelectedTextRangeAttribute` → inserted range) and **polls** it for ~30s (notifications are unreliable across apps; polling a single element is cheap and works in far more of them). When the value settles, `CorrectionDetector` (pure, self-tested via `MURMUR_TEST_CORRECTIONS`) runs a strict diff: exactly one changed word **inside our inserted range**, and the new word must both *look* learnable (ALL-CAPS acronym / letter+digit jargon / CamelCase / capitalized proper noun — suppressed at sentence start to avoid "their→There") **and** *sound* like the old one (Soundex match or Levenshtein ratio ≥0.55). Conservative by design (user's choice: names/acronyms/jargon only). A hit → `CorrectionStore` (UserDefaults JSON, `heard→corrected`), a "Learned 'X'" notch flash, and a brief **Undo** in the menu bar. Applied two ways on future dictations: exact whole-word case-insensitive replace (`CorrectionStore.apply`, runs last so it always wins) **and** the corrected spelling joins the recognizer bias prompt (merged with manual vocab in `DictationController.biasVocabulary`). New `autoLearnFromEdits` pref (default on) + a "Learned words" Settings section (toggle + reviewable/removable list). Only ever reads the field you just dictated into, time-boxed, and persists only the word pair. `xcodebuild` → BUILD SUCCEEDED; detector + replacement self-tests pass 8/8 + 4/4. Watcher itself needs a real focused field + Accessibility to exercise (not headlessly testable).
- **Whisper "infinite transcribing" — real root cause + fix** — Symptom: every Whisper model hung forever on "Transcribing…"; only Parakeet worked. Bisected empirically (env-gated diags feeding synthetic + real `say`-generated 43 s speech through the live engine): WhisperKit transcribe is fine **warm** for all models/durations/languages. The hang only reproduces **cold** — confirmed by clearing the per-app ANE cache (`~/Library/Caches/com.murmur.app/com.apple.e5rt.e5bundlecache`), after which `pipe.transcribe` stalled >5 min on an M4 Max. Two compounding bugs: (1) `WhisperService.preload()` built `WhisperKit(config)` **without `load: true`**, so it only *downloaded* (model stayed `Unloaded`) yet reported `.ready` — the heavy load + cold Core ML **ANE specialization** was deferred into the first `transcribe`, masquerading as a frozen transcription with no feedback; (2) WhisperKit defaults encoder+decoder to `.cpuAndNeuralEngine`, and that cold ANE specialization stalls indefinitely here. Parakeet (FluidAudio) actually loads in `preload()` and doesn't hit it. **Fix (one place, `WhisperService.pipeTask`):** build `ModelComputeOptions(melCompute: .cpuAndGPU, audioEncoderCompute: .cpuAndGPU, textDecoderCompute: .cpuAndGPU, prefillCompute: .cpuOnly)` + `WhisperKitConfig(model:computeOptions:load:true)`. GPU avoids the ANE compile entirely (fast on Apple Silicon) and `load: true` moves the cost into the truthful "Preparing model…" phase. Validated **cold**: 57 s end-to-end (≈37 s one-time GPU load + 18 s decode of 43 s audio); **warm**: ~11 s load + decode. `xcodebuild` → BUILD SUCCEEDED. ⚠️ The installed `~/Applications/Murmur.app` is the old build — rebuild in Xcode to get the fix.
- **Wispr-style LLM cleanup (local, llama.cpp)** — Researched: Wispr uses fine-tuned **Llama 3.1 in the cloud** (Baseten). MLX is fastest local runtime BUT conflicts with WhisperKit (incompatible swift-transformers: WhisperKit [1.1.6,1.2) vs MLX-examples <1.0 or 1.3+; no overlap). FluidAudio has zero deps. Decision (user): keep both engines + **llama.cpp via `LLM.swift` 2.1.0** (prebuilt Metal xcframework, no swift-transformers → no conflict; needs `-skipMacroValidation` on CLI / "Trust & Enable" in Xcode). Built `LLMCleaner` (actor, Qwen2.5 + ChatML, stateless `getCompletion`, low temp). `Template.llama` is Llama-2 format so used Qwen+chatML. LLM.swift's HF HTML-scrape downloader fails → wrote own resolve-URL downloader into App Support/Murmur/Models. `Preferences.smartCleanup` + `cleanupModel` (Qwen 3B default / 1.5B), `AppState.cleanupPhase`, Settings section, applied in `endRecording` ("Polishing…" notch) before deliver; supersedes filler regex. Models: tiny/base/small + quantized turbo + distil turbo + full turbo (Whisper) + Parakeet. `xcodebuild` → BUILD SUCCEEDED.
- **Transcript cleanup (fillers)** — Parakeet is verbatim (keeps um/uh, spells out numbers "twenty twenty six", keeps self-corrections). - **Speed bug + Parakeet engine** — **Root cause of "1 min for 30s":** `streamingEnabled` was ON, and the streaming loop re-transcribed the *entire growing buffer* every 0.8s (O(n²)), saturating the engine + blocking the final pass. Synthetic bench proved the engine itself is fast (warm 0.14s / ~107× RTF; the autoregressive decoder is the real cost for many-token real speech). **Fixes:** (1) streaming now previews only the last ~8s window (bounded cost); (2) Whisper decode options `withoutTimestamps=true` + `wordTimestamps=false` + `temperatureFallbackCount=1` (timestamps ~double decode steps). **Parakeet:** added FluidAudio 0.15.3 → `ParakeetService` (Parakeet TDT 0.6B v3, CoreML/ANE, non-autoregressive, multilingual). Introduced `SpeechEngine` protocol + `EngineLoadState`; `WhisperService` + `ParakeetService` conform; `DictationController` swaps engine by `Preferences.ModelChoice` (renamed from WhisperModel; added `.parakeet`, `.engine`). Removed turbo migration. `xcodebuild` → BUILD SUCCEEDED. Added a `MURMUR_BENCH` env-gated benchmark hook in `DictationController`/`AppDelegate`.
